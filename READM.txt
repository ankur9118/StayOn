StayOn

StayOn — Be Always Online

StayOn is an Android utility designed to maintain continuous internet connectivity by automatically switching between available networks when connectivity changes.

The goal of StayOn is simple:

Keep the device connected to the internet with minimal interruption.

StayOn works on Android phones and Android TV without requiring root access and remains Play Store compliant.

Project Concept

Android does not automatically switch between WiFi networks in the background.
If the current network becomes weak or disconnects, the user must manually select another network.

StayOn solves this problem by providing automatic network failover.

StayOn continuously monitors network status and automatically connects to the most appropriate network when connectivity changes.

Target Platforms

Android phones

Android TV

Android TV is a particularly strong use case because users rarely interact with network settings using a remote control.

StayOn allows the device to remain connected automatically without manual intervention.

Network Selection Logic

StayOn uses a two-layer network selection model.

1. Primary Network (User Defined)

The user can define one Primary WiFi network.

Rule:

If the Primary network is available, StayOn will always connect to it, regardless of signal strength.

Example:

Primary Network: Home Fiber

Even if another network has stronger signal strength, the device will reconnect to the Primary network whenever it becomes available.

This ensures that the device always returns to the user's preferred network.

2. Autopilot Mode

If the Primary network is not available, StayOn automatically enters Autopilot Mode.

Autopilot behavior:

Scan available WiFi networks

Filter saved networks

Evaluate signal strength (RSSI)

Select the best available network

Connect automatically

This process happens in the background without user interaction.

3. Mobile Data Fallback

If no saved WiFi networks are available, StayOn automatically falls back to:

Mobile data.

This ensures that the device remains online whenever possible.

Why Local VPN May Be Used

Android restricts automatic background WiFi switching for third-party apps.

To improve network transition handling, StayOn may use a local VPN-based routing layer.

The VPN layer can help provide:

faster failover

reduced packet loss

better background network control

This approach remains device-local and does not route traffic through external servers.
